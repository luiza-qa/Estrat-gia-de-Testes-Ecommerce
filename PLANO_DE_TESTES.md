# 📝 Plano de Testes: E-commerce "Loja Virtual QA"
**Versão:** 1.0 | **Data:** 28/02/2026 | **Responsável:** Luíza Silva

## 1. Objetivo
Garantir que o fluxo principal de compra (Happy Path) funcione sem interrupções, desde a seleção do produto até a confirmação do pagamento.

## 2. Escopo dos Testes (O que vamos testar?)
- **Funcionalidades Críticas:** Login, Busca, Carrinho e Checkout.
- **Plataformas:** Google Chrome e Microsoft Edge (Desktop).

## 3. Estratégia de Teste
- **Nível:** Teste de Sistema.
- **Tipo:** Funcional Manual e Exploratório.
- **Ferramentas:** ![Jira](https://img.shields.io/badge/Jira-0052CC?style=flat-square&logo=jirasoftware&logoColor=white) para gestão e ![Postman](https://img.shields.io/badge/Postman-FF6C37?style=flat-square&logo=postman&logoColor=white) para validação de APIs de frete.

## 4. Critérios de Aceitação
- O usuário deve conseguir finalizar uma compra em menos de 3 minutos.
- Nenhum bug de severidade "Crítica" ou "Alta" pode estar aberto para o lançamento.
